# Radar – описание проекта

Выпускной проект курса по Claude Code. Оформлен по структуре из урока: название, проблема, пользователь, основная функция, формат результата, минимальный функционал, технологии.

---

## Название

**Radar** – мультиагентный мониторинг Telegram-каналов про ИИ в праве с автопубликацией дайджеста.

---

## Проблема

Чтобы оставаться в теме «ИИ и legal-tech», юристу нужно каждый день просматривать десятки Telegram-каналов. Это 1–2 часа ручного чтения, в котором полезные сигналы тонут в шуме: рекламе, репостах, флуде и офтопе. Задача повторяется ежедневно, отнимает время и не масштабируется – при росте числа каналов читать вручную становится невозможно.

Radar снимает эту рутину: собирает сообщения из ~85 каналов, сам отсеивает шум, готовит структурированный дайджест по направлениям и публикует его в Telegram-канал. Аналитик получает готовую выжимку за день вместо часа листания ленты.

---

## Пользователь

- Практикующие юристы и адвокаты, которым важно быстро понимать, что нового на стыке ИИ и права.
- Руководители юридических департаментов – следят за трендами legal-tech и регулированием ИИ.
- Юристы-предприниматели, небольшие юрфирмы и IT-юристы.
- Ведущий Telegram-канала Radar, для которого дайджест – источник материала для постов.

---

## Основная функция

Radar – это конвейер из пяти шагов. На каждом шаге понятно, что подается на вход, что получается на выходе и какой инструмент задействован.

| Шаг | Вход | Выход | Инструмент |
|---|---|---|---|
| 1. Сбор | Список ~85 каналов (`sources/channels.json`) | Сырые сообщения за день (`data/raw/`) | Telethon (MTProto) |
| 2. Фильтрация | Сырые сообщения | Отобранные сигналы без шума (`data/processed/`) | OpenAI API |
| 3. Анализ | Отобранные сигналы | Разборы по направлениям (`data/analysis/`) | Агенты `analyst-legal`, `analyst-aitech` |
| 4. Композиция | Разборы агентов | Единый дайджест (`digests/`) | Агент `digest-composer` |
| 5. Публикация | Готовый дайджест | Пост в Telegram-канале | Telegram Bot API |

Дополнительно скилл `post-adapter` берет одну новость или фрагмент дайджеста и превращает ее в готовые посты под площадки Radar (Telegram-канал и корпоративный портал Bitrix24).

---

## Формат результата

- **Дайджест** – markdown-файл `digests/YYYY-MM-DD-digest.md` со структурой: «Главное за день», «Правовые сигналы», «ИИ и технологии», «Итог». Каждый сигнал – одной строкой со ссылкой на первоисточник.
- **Пост в Telegram-канале** – тот же дайджест, опубликованный ботом по расписанию.
- **Адаптированный пост** – JSON-файл в `posts/` с готовым текстом под Telegram и Bitrix24 (результат скилла `post-adapter`).

---

## Минимальный функционал (MVP)

Реализовано и работает:

- Сбор из ~85 Telegram-каналов по MTProto с дедупликацией.
- Фильтрация шума через OpenAI, разметка сигналов по тегам.
- Два аналитических агента по направлениям (право / ИИ-технологии) и агент-композитор.
- Оркестратор пайплайна `scripts/run-pipeline.py` с флагами `--skip-*` для частичного прогона.
- Автопубликация дайджеста в Telegram-канал через Bot API.
- Расписание на VPS через cron.
- Скилл `post-adapter` для адаптации новости под площадки.
- Скилл `radar-dev` для работы с репозиторием (структура, запуск, изменения, публикация).

Проведен полный end-to-end прогон в реальном времени – дайджест дошел до Telegram-канала.

Планы на развитие:

- Дашборд с динамикой сигналов по каналам и темам.
- Автооценка соотношения сигнал/шум по каждому источнику и авто-ревью слабых каналов.
- Расширение адаптации постов на ВКонтакте.
- Оповещения об аномалиях и срочных событиях отдельным каналом.

## Технологии

- **Язык:** Python 3.12 (venv).
- **Сбор:** Telethon (MTProto-клиент).
- **Публикация:** Telegram Bot API.
- **Фильтрация и анализ:** OpenAI API; мультиагентная оркестрация через Claude Code / облачный Routine.
- **Хранение:** файлы JSON и Markdown в репозитории; вспомогательная загрузка в SQLite (`scripts/load-to-sqlite.py`).
- **Расписание:** cron на VPS.
- **Секреты:** переменные окружения и `.env` (не в git).
- **Витрина и обмен:** GitHub-репозиторий, скиллы Claude Code (`post-adapter`, `radar-dev`).

Форматы ответов агентов – Markdown (дайджесты, разборы) и JSON (адаптированные посты, сырые и обработанные данные).

---

## Архитектура в двух звеньях

Проект развернут на двух звеньях, потому что сбор и анализ физически не могут жить в одном месте:

- **VPS** (Beget) – сбор из Telegram (MTProto) и публикация (Bot API); оба вида трафика не проходят через egress-прокси облачной песочницы.
- **Облачный Routine** (claude.ai) – мультиагентный анализ; наружу нужен только GitHub и OpenAI API.

Связующий слой – публичный GitHub-репозиторий: VPS пушит сырые данные, Routine читает их, прогоняет через агентов и коммитит дайджест обратно, VPS публикует его в канал.
