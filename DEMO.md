# Защита проекта Radar – сценарий на 3–5 минут

Структура по уроку: название и описание → проблема → живая демонстрация → результаты → GitHub → планы. Общий тайм-бокс 4 минуты. Слева – что говоришь, справа – что показываешь на экране.

## 0. Подготовка до выступления (заранее)

- Открой заранее: терминал в папке проекта, файл свежего дайджеста, Telegram-канал с опубликованным постом, страницу репозитория на GitHub.
- Сбор и публикация идут на VPS и занимают время – **не запускай их вживую**.
- `run-pipeline.py --skip-fetch` без даты работает с сегодняшним числом – если сырых данных за сегодня нет, покажет «0 signals» и выглядит как будто сломан. Для демо запускай backfill на дату, за которую данные точно есть (например, `2026-07-11`):

```bash
python scripts/filter-signals.py --date 2026-07-11
python agents/digest-agent.py --date 2026-07-11
```

Готовый дайджест держи открытым как запасной вариант.
- `filter-signals.py --date` может показать «processing 0 raw files» – это нормально, если сырые данные за эту дату уже были обработаны раньше (raw очищается после обработки, processed остается). Показательный результат – вывод `digest-agent.py`: «Loaded N signals, N total posts» и сохраненный файл дайджеста.

## Если что-то пойдет не так

- Прогон завис или упал – не жди, открой заранее подготовленный готовый дайджест: «Вот результат последнего прогона».
- Спросят про сбор/публикацию – объясни, что они идут на VPS по расписанию и в облаке недоступны из-за прокси; это осознанное архитектурное решение, а не баг.
- Спросят «почему не один сервер» – VPS не дает мультиагентной оркестрации Claude Code, а облако физически не достучится до Telegram; нужны оба звена.
- Заметят, что скриншот файла дайджеста и скриншот поста в Telegram-канале за одну и ту же дату отличаются по содержанию – это ожидаемо: дайджест пересобирался повторно (backfill) уже после исходной публикации, поэтому в файле на диске оказалась более свежая версия выборки сигналов, чем та, что ушла в канал. Показывает, что дайджест воспроизводим и его можно пересобрать в любой момент, а не единоразовый артефакт.
